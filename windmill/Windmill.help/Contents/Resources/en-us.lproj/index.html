Windmill Help

Say “Hello Windmill”.

Windmill monitors for code changes, making sure your project is in a heatlhy state every step of the way towards a release.

It does the little things during development, like making sure you haven’t forgotten to check in a file. As well as looking at the bigger picture like building your app for distribution.

Working with external dependencies.

Windmill, by default, will use the contents in the last commit that has been pushed to origin.

Should you wish to make changes in the contents before Windmill starts building a project, you can do so under Preferences. Use the menu bar, select Windmill then Preferences.

Select the Settings tab, under Advanced enable the Let me bring external dependencies checkbox. Windmill will not reset the contents to origin as part of the Checkout stage.

Important! By integrating dependencies late in the development cycle, it takes longer to get feedback on the state of your project and with an added risk of it breaking in unexpected ways. It is recommended that you install, update, integrate and keep dependencies as part of the project's source code. e.g. for CocoaPods that means checking the Pods directory into source control.

Using CocoaPods.

For a project that uses CocoaPods without checking the Pods directory into source control the Build stage will fail with a Build Failed error.

Click in the toolbar to open the Error Summary window.

You should see an error as reported by CocoaPods. Use the menu bar, select Navigate, then Show Checkout Folder In Terminal to open a terminal in the location of the project folder.

Run `$ pod install` in the project directory.

Wait for the installation to complete. Click in the toolbar to start Windmill.

Using Carthage.

For a project that uses Carthage without checking the Carthage/Build directory into source control the Build stage will fail with a Build Failed error.

Click in the toolbar to open the Error Summary window.

You should see an error as reported by Carthage. Use the menu bar, select Navigate, then Show Checkout Folder In Terminal to open a terminal in the location of the project folder.

Run `$ carthage bootstrap --platform iOS` in the project directory.

Wait for the installation to complete. Click in the toolbar to start Windmill.
